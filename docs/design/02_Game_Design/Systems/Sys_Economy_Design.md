> **文档版本**：v2.1.0

## 一、经济系统概述

### 1.1 设计目标

《徒踪-V2.0》的经济系统需要同时满足休闲探索模式和生存挑战模式的需求，因此采用**三货币体系**：

**核心目标**：

1. 支持两种模式的独特玩法需求
2. 鼓励长期养成和持续游玩
3. 提供多样化的消费和成长路径
4. 保持经济系统的平衡和可持续性

### 1.2 三货币体系

| 货币名称 | 定位 | 获取难度 | 主要用途 |
| --- | --- | --- | --- |
| **环保值**
Eco Points | 关卡内流通货币 | 容易 | 购买消耗品、装备、道具 |
| **爬升值**
Climbing Points | 长期养成货币 | 困难 | 稀有装备、永久提升、解锁内容 |
| **徒步数**
Hiking Distance | 里程积累货币 | 中等 | 装备升级、路线解锁 |

## 二、货币系统详解

### 2.1 环保值（Eco Points）

**货币定位**：日常流通货币，用于关卡内和关卡间的短期消费

#### 2.1.1 获取渠道

**休闲探索模式**：

| 获取方式 | 获取量 | 频率 | 备注 |
| --- | --- | --- | --- |
| 完成关卡基础奖励 | 50-150 | 每关 | 根据层数递增 |
| 消除环保卡 | 10-30/张 | 关卡内 | 主动触发 |
| 完成环保行动 | 20-50/次 | 关卡内 | 特殊事件 |
| 连击奖励 | 5×连击数 | 关卡内 | 高连击额外奖励 |
| 完美通关 | +50 | 每关 | 体能>80% |
| 照片卡收集 | 30-100/张 | 收集时 | 按稀有度 |

**单局期望收益**：300-500环保值（15分钟游玩时长）

**生存挑战模式**：

| 获取方式 | 获取量 | 频率 | 备注 |
| --- | --- | --- | --- |
| 完成关卡基础奖励 | 100-250 | 每关 | 根据难度递增 |
| 生存天数奖励 | 50/天 | 关卡内 | 每度过1天 |
| 完成环保行动 | 30-80/次 | 关卡内 | 难度更高 |
| 极端天气生存 | 100-200 | 关卡内 | 成功度过极端天气 |
| 完美通关 | +100 | 每关 | 所有属性>60% |
| 挑战成就 | 50-300 | 达成时 | 特殊挑战 |

**单局期望收益**：500-800环保值（25分钟游玩时长）

#### 2.1.2 消费渠道

**关卡内消费（游动商贩）**：

| 商品类型 | 价格区间 | 购买频率 |
| --- | --- | --- |
| 矿泉水（口渴+20） | 25环保值 | 高频 |
| 能量棒（饥饿+20） | 30环保值 | 高频 |
| 急救喷雾（体能+15） | 50环保值 | 中频 |
| 修复工具（耐久度+100%） | 80环保值 | 低频 |
| 临时增益道具 | 60-70环保值 | 中频 |

**关卡后消费（景区商店）**：

| 商品类型 | 价格区间 | 性价比 |
| --- | --- | --- |
| 消耗品组合包 | 60-90环保值 | 高 |
| 普通装备 | 150-250环保值 | 中 |
| 稀有装备 | 400-600环保值 | 低 |
| 照片卡包 | 100-500环保值 | 中 |
| 经验书 | 150环保值 | 中 |

#### 2.1.3 经济循环

```
玩家游玩关卡 
    ↓
获得300-800环保值
    ↓
┌────────────────────────────┐
│ 消费选择：                  │
│ A. 购买消耗品（25-80）      │
│ B. 购买装备（150-600）      │
│ C. 购买照片卡包（100-500）  │
│ D. 存储等待大额消费         │
└────────────────────────────┘
    ↓
继续游玩 → 持续循环
```

**设计目标**：

- 玩家每局能买1-2件消耗品
- 每3-5局能买1件普通装备
- 每10-15局能买1件稀有装备

### 2.2 爬升值（Climbing Points）

**货币定位**：稀缺养成货币，用于重要的永久性提升

#### 2.2.1 获取渠道

**两种模式通用**：

| 获取方式 | 获取量 | 频率 | 备注 |
| --- | --- | --- | --- |
| 通关基础奖励 | 关卡层数÷2 | 每关 | 5层关卡=2-3爬升值 |
| 完美通关奖励 | +5 | 每关 | 高质量通关 |
| 极限挑战奖励 | +10 | 挑战时 | 仅挑战模式 |
| 成就奖励 | 10-100 | 达成时 | 重要成就 |

**日均期望收益**（纯关卡通关）：

- 轻度玩家（1小时/天）：10-15爬升值
- 中度玩家（2小时/天）：25-35爬升值
- 重度玩家（4小时/天）：50-70爬升值

#### 2.2.2 消费渠道

**局外商店（永久提升）**：

| 提升项目 | 效果 | 价格 | 最大等级 |
| --- | --- | --- | --- |
| 体能扩展 | 体能上限+10 | 20爬升值（每级+10） | 10级 |
| 负重增强 | 负重上限+2kg | 25爬升值（每级+15） | 8级 |
| 装备槽扩展 | +1装备槽位 | 50/100/200爬升值 | 3级 |
| 照片卡加成 | 照片卡效果+5% | 30爬升值（每级+20） | 5级 |
| 起始资源 | 开局环保值+20 | 15爬升值（每级+10） | 10级 |
| 经验加成 | 经验获取+10% | 40爬升值（每级+30） | 5级 |

**局外商店（稀有装备）**：

| 装备类型 | 价格 | 库存 |
| --- | --- | --- |
| 碳纤维登山杖（稀有） | 50爬升值 | 每周5个 |
| 专业登山靴（稀有） | 60爬升值 | 每周5个 |
| 户外探险背包（稀有） | 70爬升值 | 每周5个 |
| GPS导航仪（史诗） | 120爬升值 | 每周2个 |
| 急救包（史诗） | 150爬升值 | 每周2个 |
| 传说装备宝箱 | 300爬升值 | 每月1个 |

**特殊解锁**：

| 解锁内容 | 价格 | 说明 |
| --- | --- | --- |
| 角色解锁券 | 100爬升值 | 直接解锁未解锁角色 |
| 隐藏路线解锁 | 80-150爬升值 | 解锁特殊挑战路线 |
| 皮肤解锁 | 50-200爬升值 | 角色外观皮肤 |

#### 2.2.3 消费节奏

**新手期（前20小时）**：

- 主要消费：角色解锁（100）、基础永久提升（200-300）
- 累计消费：300-400爬升值

**成长期（20-100小时）**：

- 主要消费：装备槽扩展（350）、稀有装备（200-300）
- 累计消费：800-1200爬升值

**成熟期（100小时+）**：

- 主要消费：高级永久提升（500+）、传说装备（300+）
- 持续消费：每周100-200爬升值

### 2.3 徒步数（Hiking Distance）

**货币定位**：里程积累货币，衡量玩家游玩时长和投入

#### 2.3.1 获取渠道

| 获取方式 | 获取量 | 说明 |
| --- | --- | --- |
| 消除/穿越卡牌 | 10-50/张 | 根据地形类型 |
| 完成关卡 | 100 + 层数×20 | 基础奖励 |
| 照片卡收集 | 200-800/张 | 按稀有度 |
| 成就达成 | 500-5000 | 里程碑成就 |

**日均期望收益**（纯关卡游玩）：

- 轻度玩家：2,000-3,000徒步数
- 中度玩家：5,000-8,000徒步数
- 重度玩家：10,000-15,000徒步数

#### 2.3.2 消费渠道

**装备升级系统**：

| 升级等级 | 所需徒步数 | 额外材料 | 效果提升 |
| --- | --- | --- | --- |
| 1→2级 | 1,000 | 1个同类装备 | +10% |
| 2→3级 | 4,000 | 2个同类装备 | +10% |
| 3→4级 | 9,000 | 3个同类装备 | +10% |
| 4→5级 | 16,000 | 5个同类装备 | +10% |
| 总计1→5级 | 30,000 | 11个同类装备 | +40% |

**路线解锁**：

| 路线类型 | 所需徒步数 | 解锁条件 |
| --- | --- | --- |
| 基础路线 | 0 | 初始可用 |
| 进阶路线 | 5,000-10,000 | 通关基础路线 |
| 高级路线 | 15,000-25,000 | 通关进阶路线 |
| 隐藏路线 | 30,000-50,000 | 特殊成就+徒步数 |

**其他消费**：

| 消费项目 | 所需徒步数 | 说明 |
| --- | --- | --- |
| 装备修理（全恢复） | 500-2000 | 根据装备等级 |
| 图鉴完成奖励查看 | 1,000 | 查看已收集内容统计 |
| 个人数据面板美化 | 5,000-20,000 | 外观定制 |

## 三、资源获取与消耗平衡

### 3.1 每日游玩循环

**典型玩家（2小时/天）游玩流程**：

```jsx
10:00 休闲模式游玩（3关，45分钟）
  • 环保值：+1,200（400×3）
  • 爬升值：+15（基础+完美）
  • 徒步数：+4,500

  消费：
  • 游动商贩购买消耗品：-150环保值
  • 景区商店购买装备：-400环保值
  • 剩余：650环保值

11:00 挑战模式游玩（2关，50分钟）
  • 环保值：+1,400（700×2）
  • 爬升值：+25（基础+完美+挑战）
  • 徒步数：+5,000

  消费：
  • 游动商贩购买消耗品：-200环保值
  • 景区商店购买稀有装备：-500环保值
  • 剩余：700环保值

12:00 局外活动（25分钟）
  • 查看商店，购买永久提升：-30爬升值
  • 使用徒步数升级装备：-4,000徒步数

当日总结：
  • 环保值：+1,350（收支相抵后积累）
  • 爬升值：+10（收支相抵后积累）
  • 徒步数：+5,600（收支相抵后积累）
```

### 3.2 货币积累曲线

**环保值积累**：

| 游玩阶段 | 累计环保值 | 主要用途 |
| --- | --- | --- |
| 第1周（10小时） | 5,000-8,000 | 购买消耗品、基础装备 |
| 第1月（40小时） | 25,000-35,000 | 购买稀有装备、照片卡包 |
| 第3月（120小时） | 80,000-120,000 | 持续消费，积累稳定 |

**爬升值积累**：

| 游玩阶段 | 累计爬升值 | 主要消费 |
| --- | --- | --- |
| 第1周（10小时） | 80-120 | 解锁角色、基础提升 |
| 第1月（40小时） | 400-550 | 装备槽扩展、稀有装备 |
| 第3月（120小时） | 1,500-2,200 | 高级提升、传说装备 |

**徒步数积累**：

| 游玩阶段 | 累计徒步数 | 主要消费 |
| --- | --- | --- |
| 第1周（10小时） | 25,000-35,000 | 装备升级1-2级 |
| 第1月（40小时） | 120,000-160,000 | 装备升级3-4级、路线解锁 |
| 第3月（120小时） | 400,000-550,000 | 装备升级满级、全路线解锁 |

### 3.3 经济平衡检查点

**检查点1：新手期（前5小时）**

- 目标：玩家能体验完整游戏循环
- 检查：是否有足够环保值购买消耗品？
- 标准：玩家不会因为缺少资源而卡关

**检查点2：成长期（5-50小时）**

- 目标：玩家感受到明显成长
- 检查：是否能解锁新角色、购买稀有装备？
- 标准：每10小时有明显进步

**检查点3：成熟期（50小时+）**

- 目标：维持长期游玩动力
- 检查：是否还有有意义的成长目标？
- 标准：仍有稀缺资源和解锁内容

## 四、交易系统

### 4.1 商店系统

#### 游动商贩（关卡内）

- **刷新**：随机出现，概率10%
- **商品**：基础消耗品为主
- **价格**：随游玩次数微调（+5%/5次）

#### 景区商店（山顶后）

- **刷新**：每关通关后1次
- **商品**：消耗品+装备+照片卡包
- **折扣**：根据通关表现提供折扣（最低7折）

#### 局外商店（主界面）

- **刷新**：每周一自动刷新
- **商品**：稀有装备+永久提升
- **手动刷新**：10爬升值/次，每周最多3次

### 4.2 兑换系统

**货币不可互相兑换**，但可以通过游玩获取：

- 不提供环保值→爬升值兑换（防止刷货币）
- 不提供爬升值→环保值兑换（保持稀缺性）
- 不提供徒步数→其他货币兑换（里程积累）

## 五、成长机制

### 5.1 角色成长

**经验系统**：

- 经验获取与货币获取同步
- 升级后获得属性点和技能点
- 不消耗货币，纯时间投入

**属性提升**：

- 体能上限：20爬升值/级，最多10级
- 负重上限：25爬升值/级，最多8级
- 恢复速率：30爬升值/级，最多5级

### 5.2 装备成长

**装备获取**：

- 环保值购买（150-600）
- 爬升值购买（50-300）
- 通关奖励
- 照片卡收集奖励

**装备升级**：

- 消耗徒步数（1,000-16,000）
- 消耗同类装备作为材料
- 每级提升10%效果

### 5.3 收集成长

**照片卡收集**：

- 休闲模式主要获取途径
- 收集奖励：环保值、徒步数、特殊装备
- 图鉴完成度影响成就解锁

## 六、经济调控机制

### 6.1 通货膨胀控制

**环保值控制**：

- 主要消耗：消耗品（快速消耗）
- 次要消耗：装备（中期消耗）
- 存储上限：无上限，但鼓励消费

**爬升值控制**：

- 获取速率稳定
- 消费选项丰富
- 长期目标持续更新

### 6.2 价格调整机制

**动态价格**：

```
func get_shop_price(base_price: int, play_count: int) -> int:
    var multiplier = 1.0 + (play_count / 20) * 0.05
    return int(base_price * multiplier)
```

**折扣系统**：

- 完美通关：9折
- 全照片卡收集：9折
- 全环保行动：8.5折
- 多条件叠加：最低7折

### 6.3 防刷机制

**操作限制**：

- 商店手动刷新：最多3次/周

**数据完整性**：

- 本地存档加密（JSON+AES）
- 关键数据校验和验证

## 七、实现建议

### 7.1 Godot数据结构

**货币管理器**：

```
class_name CurrencyManager extends Node

# 货币数据
var eco_points: int = 0        # 环保值
var climbing_points: int = 0   # 爬升值  
var hiking_distance: int = 0   # 徒步数

# 货币变化信号
signal currency_changed(currency_type: String, old_value: int, new_value: int)

# 增加货币
func add_currency(type: String, amount: int) -> void:
    var old_value = get_currency(type)
    match type:
        "eco":
            eco_points += amount
        "climbing":
            climbing_points += amount
        "hiking":
            hiking_distance += amount
    
    var new_value = get_currency(type)
    currency_changed.emit(type, old_value, new_value)
    
    # 保存数据
    save_currency_data()

# 消耗货币
func spend_currency(type: String, amount: int) -> bool:
    if not has_enough_currency(type, amount):
        return false
    
    add_currency(type, -amount)
    return true

# 检查货币是否足够
func has_enough_currency(type: String, amount: int) -> bool:
    return get_currency(type) >= amount
```

### 7.2 商店系统实现

```
class_name ShopSystem extends Node

# 商店类型
enum ShopType { WANDERING, SCENIC, PERMANENT }

# 生成商店商品
func generate_shop_products(shop_type: ShopType) -> Array:
    var products = []
    
    match shop_type:
        ShopType.WANDERING:
            products = generate_wandering_products()
        ShopType.SCENIC:
            products = generate_scenic_products()
        ShopType.PERMANENT:
            products = generate_permanent_products()
    
    return products

# 购买商品
func buy_product(product: ShopProduct) -> bool:
    # 检查货币
    if not CurrencyManager.has_enough_currency(product.currency_type, product.price):
        return false
    
    # 扣除货币
    if not CurrencyManager.spend_currency(product.currency_type, product.price):
        return false
    
    # 发放商品
    PlayerInventory.add_item(product.item_id, product.quantity)
    
    return true
```

## 八、常见问题（FAQ）

### Q1：为什么设计三种货币而不是一种？

A：三种货币满足不同层次的需求：环保值用于日常消费，爬升值用于长期成长，徒步数衡量投入时间。这样可以给玩家更多策略选择。

### Q2：爬升值会不会太难获得？

A：爬升值的获取速率经过精心设计，轻度玩家每天也能获得15-25个，足够进行有意义的提升。

### Q3：两种模式的货币获取差异会导致玩家只玩一种模式吗？

A：挑战模式收益更高，但难度也更大。

### Q4：徒步数有上限吗？

A：没有上限。徒步数是里程积累的象征，持续增长。

### Q5：如果货币经济失衡怎么办？

A：我们设计了动态价格调整和定期数据分析机制，可以通过配置文件快速调整各项数值。
