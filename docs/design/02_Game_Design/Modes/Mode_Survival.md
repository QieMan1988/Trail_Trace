> **文档版本**：v2.1.0

## 一、模式概述

### 1.1 设计定位

生存挑战模式是《徒踪-V2.0》面向硬核玩家的挑战性玩法，以**全国高风险徒步路线**为背景，通过严苛的生存管理和极端天气挑战，为玩家提供真实的户外生存体验。

**核心特点**：

- ⚠️ **真实风险**：基于真实户外事故案例设计
- 🌡️ **严苛环境**：极端天气、复杂地形
- 🧠 **策略决策**：资源有限，决策至关重要
- 🏆 **高回报**：通关奖励远超休闲模式

### 1.2 目标玩家

- **硬核玩家**：喜欢高难度挑战和风险管理
- **策略向玩家**：享受决策和资源分配的乐趣
- **户外爱好者**：对真实徒步和登山感兴趣
- **挑战向玩家**：追求极限和自我突破

### 1.3 游玩时长

- **单局时长**：20-30分钟
- **单日推荐**：2-4局（40-120分钟）
- **完整通关**：30-50小时

## 二、5属性管理系统

### 2.1 属性设计

生存挑战模式采用**5属性系统**，用**心智**替代休闲模式的心率：

| 属性名称 | 取值范围 | 初始值 | 消耗速度 | 危险阈值 |
| --- | --- | --- | --- | --- |
| **体能**
Stamina | 0-100 | 100 | 快 | <15 |
| **饥饿**
Hunger | 0-100
(100=饱) | 100 | 快 | <20 |
| **口渴**
Thirst | 0-100
(100=不渴) | 100 | 极快 | <15 |
| **体温**
Temperature | 32-42°C | 36.5°C | 取决于天气 | <34 或 >40 |
| **心智**
Sanity | 0-100 | 100 | 中 | <30 |

### 2.2 属性消耗机制

#### 2.2.1 体能消耗

**消耗规则**：

- 每前进1个单位距离消耗体能
- 消耗量 = 基础消耗 × 地形系数 × 负重系数 × 天气系数

| 地形类型 | 基础消耗 | 地形系数 | 实际消耗 |
| --- | --- | --- | --- |
| 平地 | 5 | 1.0 | 5体能 |
| 缓坡 | 5 | 1.5 | 8体能 |
| 陡坡 | 5 | 2.0 | 10体能 |
| 碎石路 | 5 | 2.2 | 11体能 |
| 峭壁 | 5 | 3.0 | 15体能 |
| 冰雪地 | 5 | 2.5 | 13体能 |

**天气系数**：

- 晴天：1.0
- 阴天：1.1
- 小雨：1.3
- 暴雨：1.8
- 大雪：2.0
- 暴风雪：2.5

#### 2.2.2 饥饿消耗

**消耗规则**：

- 每前进10个单位：饥饿值-1
- 每度过1天（游戏内时间）：饥饿值-15
- 极寒天气（<-10°C）：饥饿消耗×1.5

**饥饿影响**：

| 饥饿值 | 状态 | 影响 |
| --- | --- | --- |
| 70-100 | 饱腹 | 无影响 |
| 40-69 | 正常 | 无影响 |
| 20-39 | 饥饿 | 体能消耗+20%，体能恢复-30% |
| 0-19 | 极度饥饿 | 体能消耗+40%，体能恢复停止，心智-5/分钟 |

#### 2.2.3 口渴消耗

**消耗规则**：

- 每前进5个单位：口渴值-1
- 每度过1天：口渴值-20
- 炎热天气（>30°C）：口渴消耗×2.0
- 高海拔（>3000m）：口渴消耗×1.3

**口渴影响**：

| 口渴值 | 状态 | 影响 |
| --- | --- | --- |
| 50-100 | 充足 | 无影响 |
| 20-49 | 口渴 | 体能消耗+25%，心智-2/分钟 |
| 0-19 | 严重缺水 | 体能消耗+50%，体能恢复-50%，心智-8/分钟 |

#### 2.2.4 体温变化

**变化规则**：

- 受环境温度影响：每分钟向环境温度靠近0.5°C
- 受装备影响：保暖装备减缓体温下降
- 受运动影响：剧烈运动时体温+0.2°C/分钟

**环境温度**：

| 天气 | 海拔 | 环境温度 |
| --- | --- | --- |
| 晴天 | <1000m | 25°C |
| 晴天 | 1000-3000m | 15°C |
| 晴天 | >3000m | 5°C |
| 暴风雪 | >3000m | -20°C |

**体温影响**：

| 体温范围 | 状态 | 影响 |
| --- | --- | --- |
| 36-38°C | 正常 | 无影响 |
| 38-40°C | 发热 | 口渴消耗+50%，心智-3/分钟 |
| >40°C | 中暑 | 体能-10/分钟，心智-10/分钟 |
| 34-36°C | 体温偏低 | 体能恢复-40%，饥饿消耗+30% |
| <34°C | 失温 | 体能-5/分钟，心智-8/分钟 |

#### 2.2.5 心智消耗

**消耗规则**：

- 遭遇极端天气：心智-10
- 夜间行进：心智-2/分钟
- 孤独感（单独行动>2天）：心智-5/天
- 受伤：心智-15
- 目睹危险（悬崖、雪崩等）：心智-8

**心智影响**：

| 心智值 | 状态 | 影响 |
| --- | --- | --- |
| 70-100 | 正常 | 无影响 |
| 50-69 | 紧张 | 决策时间+20% |
| 30-49 | 焦虑 | 决策时间+50%，体能消耗+15% |
| 0-29 | 恐慌 | 随机错误决策概率30%，体能消耗+30% |

**心智恢复**：

- 扎营休息：+10/小时
- 与NPC队友对话：+15
- 欣赏美景（触发事件）：+20
- 完成阶段目标：+25

### 2.3 属性恢复

**自然恢复**：

- 体能：休息时每分钟+5（站立），+10（坐下），+15（躺下扎营）
- 饥饿：不可自然恢复
- 口渴：不可自然恢复
- 体温：向环境温度靠近
- 心智：扎营休息时+10/小时

**道具恢复**：

| 道具名称 | 恢复效果 | 重量 | 耐久 |
| --- | --- | --- | --- |
| 矿泉水（500ml） | 口渴+25 | 0.5kg | 一次性 |
| 能量棒 | 饥饿+15，体能+10 | 0.1kg | 一次性 |
| 压缩饼干 | 饥饿+30 | 0.2kg | 一次性 |
| 急救包 | 体能+30，心智+15 | 0.5kg | 3次 |
| 保温毯 | 体温向36.5°C恢复 | 0.3kg | 永久 |
| 咖啡 | 心智+20，体温+1°C | 0.2kg | 一次性 |

## 三、生存决策系统

### 3.1 决策类型

游戏过程中会不断出现需要玩家决策的事件：

#### 3.1.1 路线选择

**场景**：前方路线分叉

**选项示例**：

1. **主路**（安全但远）
    - 距离：+500m
    - 体能消耗：+50
    - 风险：低
2. **捷径**（近但危险）
    - 距离：+200m
    - 体能消耗：+30
    - 风险：中（30%遭遇意外）
3. **探索路径**（未知）
    - 距离：未知
    - 可能发现宝箱或陷阱
    - 风险：高

#### 3.1.2 天气应对

**场景**：突然遭遇暴风雪

**选项示例**：

1. **原地扎营**
    - 消耗：-1天时间
    - 资源消耗：食物×2，水×2
    - 安全：高
2. **冒险前进**
    - 天气系数×2.0
    - 心智-15
    - 风险：40%失温
3. **寻找庇护所**
    - 消耗：1小时，体能-20
    - 成功率：60%
    - 成功：找到庇护所，安全度过
    - 失败：体能额外-20，被迫扎营

#### 3.1.3 资源分配

**场景**：补给不足，需要决定优先保障哪个属性

**选项示例**：

1. **节省食物**
    - 每餐饥饿恢复-50%
    - 延长补给时间×1.5
2. **节省水源**
    - 每次饮水口渴恢复-50%
    - 延长补给时间×1.3
3. **正常消耗**
    - 维持当前状态
    - 可能提前耗尽补给

#### 3.1.4 紧急事件

**事件类型**：

- 装备损坏：修理 vs 丢弃 vs 凑合用
- 遇到野生动物：躲避 vs 驱赶 vs 绕行
- 发现受伤驴友：救助 vs 报警 vs 离开
- 迷路：原路返回 vs 继续前进 vs 等待救援

### 3.2 决策影响

**即时影响**：

- 属性变化（体能、口渴、饥饿等）
- 资源消耗（食物、水、装备耐久）
- 时间推进（白天/黑夜）

**长期影响**：

- 路线进度
- 队友关系（如果有NPC队友）
- 特殊事件触发
- 成就解锁

## 四、极端天气系统

### 4.1 天气类型

| 天气类型 | 出现概率 | 主要影响 | 应对策略 |
| --- | --- | --- | --- |
| 晴天 | 40% | 无 | 正常前进 |
| 阴天 | 20% | 体能消耗+10% | 正常前进 |
| 小雨 | 15% | 体能消耗+30%，体温-1°C/10分钟 | 穿雨衣，加快速度 |
| 暴雨 | 10% | 体能消耗+80%，体温-3°C/10分钟，能见度低 | 建议扎营 |
| 大雪 | 8% | 体能消耗+100%，体温-5°C/10分钟 | 保暖，扎营 |
| 暴风雪 | 5% | 体能消耗+150%，体温-10°C/10分钟，心智-10/10分钟 | 必须扎营 |
| 极端高温 | 2% | 口渴消耗×3，体温+2°C/10分钟 | 寻找阴凉，补水 |

### 4.2 天气预警

**预警系统**：

- 提前10-15分钟显示天气变化预警
- 玩家可以选择：加速前进、原地扎营、寻找庇护所

**预警准确度**：

- 基础准确度：70%
- GPS导航仪：+15%
- 气压计：+10%
- 经验丰富天赋：+5%

### 4.3 天气特殊事件

**雷暴**：

- 触发条件：暴雨天气
- 影响：心智-20，可能击中金属装备
- 应对：摘下金属装备，蹲下避雷

**雪崩**：

- 触发条件：大雪天气+陡坡地形
- 影响：强制后退，体能-40，可能掉落装备
- 应对：快速逃离危险区域

**沙尘暴**（沙漠路线）：

- 影响：能见度为0，被迫停止前进
- 应对：扎营等待，遮盖口鼻

## 六、装备与负重系统

### 6.1 装备分类

| 装备类别 | 作用 | 重量范围 |
| --- | --- | --- |
| 背包 | 提升负重上限 | 1.5-3.0kg |
| 登山鞋 | 减少地形消耗 | 0.8-1.2kg |
| 登山杖 | 减少体能消耗 | 0.3-0.6kg |
| 帐篷 | 扎营必需 | 1.5-3.5kg |
| 睡袋 | 保温，扎营休息效果+50% | 1.0-2.0kg |
| 冲锋衣 | 防风防雨，减缓体温下降 | 0.5-0.8kg |
| 羽绒服 | 保暖，减缓体温下降 | 0.6-1.0kg |
| GPS导航仪 | 防迷路，天气预警 | 0.2kg |
| 头灯 | 夜间照明 | 0.1kg |

### 6.2 负重系统

**负重上限**：

- 基础：15kg
- 装备加成：背包+5-10kg
- 角色加成：力量型角色+3kg
- 照片卡加成：最多+5kg

**负重影响**：

```jsx
当前负重 ≤ 负重上限：无影响
当前负重 > 负重上限：体能消耗 × (1 + 超重量/10)

例：负重20kg，上限15kg
体能消耗 = 基础消耗 × (1 + 5/10) = 基础消耗 × 1.5
```

**装备耐久**：

- 所有装备都有耐久度
- 使用过程中逐渐降低
- 耐久<50%：效果减半
- 耐久=0：装备损坏，失去效果
- 可以使用修理工具恢复耐久

## 七、扎营与休息系统

### 7.1 扎营机制

**扎营条件**：

- 拥有帐篷
- 相对平坦的地形
- 非极端天气（暴风雪除外，此时必须扎营）

**扎营效果**：

- 体能恢复：+15/小时（有睡袋）或+10/小时（无睡袋）
- 心智恢复：+10/小时
- 体温：向睡袋保暖温度靠近
- 时间推进：每次扎营=8小时

**扎营风险**：

- 消耗食物×2，水×2
- 时间推进（可能错过窗口期）
- 野生动物袭击（5%概率）

### 7.2 休息策略

**短暂休息**（5-10分钟）：

- 体能恢复少量
- 不消耗食物和水
- 不推进时间

**扎营休息**（8小时）：

- 体能和心智大幅恢复
- 消耗食物和水
- 时间推进（白天→夜晚 或 夜晚→白天）

## 九、常见问题（FAQ）

### Q1：生存挑战模式和休闲模式最大的区别是什么？

A：核心区别是**心智**属性替代**心率**。心智系统让玩家不仅要管理身体状态，还要管理精神状态，增加了策略深度。此外，属性消耗速度更快，容错率更低（30% vs 70%）。

### Q2：如何应对极端天气？

A：1) 注意天气预警，提前准备；2) 携带足够的保暖/防雨装备；3) 必要时选择扎营等待；4) 升级GPS提高预警准确度。

### Q3：心智值降低到危险阈值会怎样？

A：心智<30时，你的决策可能出现随机错误（30%概率）。例如选择了A选项，系统可能执行B选项。这模拟了恐慌状态下的非理性决策。

### Q4：是否可以不带帐篷？

A：理论上可以，但极不推荐。无法扎营意味着你必须在一天内完成整个路线，而高风险路线通常需要3-7天。

### Q5：如何获得A级评价？

A：需要优秀的资源管理和路线规划。建议：1) 选择最优路径；2) 精确计算资源消耗；3) 避免不必要的风险决策；4) 利用装备减少消耗。

### 10.2 核心脚本示例

**生存属性管理器**：

```
class_name SurvivalAttributeManager extends Node

var stamina: float = 100.0
var hunger: float = 100.0
var thirst: float = 100.0
var temperature: float = 36.5
var sanity: float = 100.0

func update_attributes(delta: float, weather: Weather, terrain: String):
    # 体能消耗
    var stamina_cost = calculate_stamina_cost(terrain, weather)
    stamina -= stamina_cost * delta
    
    # 口渴消耗
    var thirst_cost = 0.2 * weather.thirst_multiplier
    thirst -= thirst_cost * delta
    
    # 饥饿消耗
    hunger -= 0.1 * delta
    
    # 体温变化
    var temp_change = (weather.environment_temp - temperature) * 0.01
    temperature += temp_change * delta
    
    # 心智消耗
    if weather.type == Weather.EXTREME:
        sanity -= 0.05 * delta
    
    # 检查危险状态
    check_danger_states()
```

**天气系统**：

```
class_name WeatherManager extends Node

enum WeatherType { SUNNY, CLOUDY, RAIN, STORM, SNOW, BLIZZARD, HEAT }

var current_weather: WeatherType = WeatherType.SUNNY
var next_weather: WeatherType
var weather_change_timer: float = 0

func _process(delta):
    weather_change_timer += delta
    
    # 每10-20分钟可能改变天气
    if weather_change_timer > randf_range(600, 1200):
        change_weather()
        weather_change_timer = 0

func change_weather():
    # 提前10分钟预警
    next_weather = get_random_weather()
    await get_tree().create_timer(600).timeout
    current_weather = next_weather
    weather_changed.emit(current_weather)
```
